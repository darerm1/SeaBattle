# SeaBattle
Сетевая многопользовательская игра в морской бой. Два игрока могут играть на разных устройствах, подключаясь к общему серверу.

### Функциональные требования
1. Сервер долже уметь принимать одновременные TCP-подключения от клиентов на указанном порту и корректно обрабатывать отключение клиента.
2. Пользователь должен иметь возможность авторизоваться.
3. Игроки, готовые к игре, помещаются в общую очередь. Подбор соперника осуществляется по поиску ближайшего рейтинга. При этом до начала игры пользователь может выйти из поиска.
4. При подборе двух игроков создаётся игровая локация, которая находится в отдельном потоке и существует до конца игры.
5. К приложению привязана база данных для хранения пользователей и их рейтинга. Обращения к базе данных работают асинхронно в отдельных потоках, не блокируя игровой процесс.

### Логика игры
1. Перед началом игроки расставляют свои корабли на поле по стандартным правилам.
2. Ход совершается по очереди. Первым стартует игрок с меньшим рейтингом. В случае одинаковых рейтингов первым стартует игрок, который находился первее в очереди на момент подбора пары.
3. При совершении хода определяется его валидность. Попадание или промах отмечаются определёным образом.
4. Выигрывает пользователь, затопивший все корабли противника первым. Рейтинг победителя увеличивается на 10% от разницы рейтинков игроков, то есть на $\dfrac{|r_в - r_п| \cdot 10}{100}$ очков, где $r_в$ - рейтинг выигравшего игрока, $r_п$ - рейтинг проигравшего. Рейтинг проигравшего уменьшается на такое же число очков.
5. При отключении игрока второй объявляется победителем, игра сохраняется. Если игрок не делает ход 30 секунд, он считается проигравшим.

### Клиентская часть
1. У пользователя должно быть окно подключения для ввода адреса сервера, порта, никнейма.
2. У пользователя отображается своё поле и поле противника. На собственном поле отображены корабли и ходы противника. На своём поле отображаются собственные ходы и попадания в корабли соперника. 
3. Сбоку отображается оставшееся количество кораблей у игрока и у противника, также отображается, чей сейчас ход.
4. После окончаения игры показывается окно с изменением рейтинга
